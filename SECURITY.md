# Security Policy

PCIS is an integrity layer for AI agents — security is the product, not an
afterthought. Reports are taken seriously and answered promptly.

## Reporting a vulnerability

Please **do not** open a public issue for security problems.

Email **idwty@proton.me** with:

- a description of the issue and its impact,
- steps to reproduce (a proof-of-concept if you have one),
- any suggested remediation.

Expect an acknowledgement within a few days. We'll work with you on a fix and
coordinate disclosure timing, and we're glad to credit reporters who want it.

## Scope

This policy covers the code in this repository. PCIS is a v1.x project with
known, documented gaps (see `CONTRIBUTING.md`) — reports that sharpen the threat
model around its cryptographic and provenance claims are especially welcome.

## Supported versions

The `main` branch receives security fixes. Tagged releases are point-in-time
snapshots; pull `main` for the latest.
